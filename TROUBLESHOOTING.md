# HoneyPot v3 Troubleshooting

Use this runbook when setup or operations do not work as expected. Commands assume you are in the HoneyPot project directory.

## Docker is not installed or not running

Check:

```bash
docker --version
docker compose version
```

Fix: install Docker Desktop or Docker Engine, start Docker, then retry `./scripts/quick_deploy.sh docker`.

## Health check failed

Check:

```bash
docker compose ps
docker compose logs -f honeypot
curl -i http://127.0.0.1:5050/api/health
```

Expected: container is `Up`/healthy and `/api/health` returns HTTP 200 JSON.

Common fixes:

- regenerate `.env` with `python setup.py` or `./scripts/quick_deploy.sh docker`;
- fix weak/default production secrets;
- check port conflicts;
- confirm Docker volumes are writable.

## Port already in use

Check:

```bash
lsof -i :5050
lsof -i :8080
lsof -i :2222
```

Fix: stop the conflicting process or change the relevant dashboard/sensor port before deployment.

## Cannot log in

Check:

```bash
test -f .deploy-credentials.txt && sed -n '1,5p' .deploy-credentials.txt
python - <<'PY'
from pathlib import Path
print('.env exists:', Path('.env').exists())
PY
```

Fix:

- use the admin account generated by `setup.py` or `scripts/quick_deploy.sh`;
- save credentials in a password manager;
- delete `.deploy-credentials.txt` after saving;
- if credentials are lost, regenerate safely with `setup.py` after backing up `.env` and the database.

## Dashboard is not reachable from another machine

This is expected by default. Docker publishes the dashboard/API on `127.0.0.1:5050` so it is private on the host.

Safe access options:

- SSH tunnel;
- Tailscale or VPN;
- Cloudflare Access;
- reverse proxy with TLS plus IP allowlist or SSO.

Do not publish `5050` directly to the public internet.

## Sensors are not receiving traffic

Check:

```bash
docker compose ps
curl -i http://127.0.0.1:8080/.env
curl -i http://SERVER_IP:8080/.env
```

Fix:

- verify cloud firewall/security group rules;
- verify host firewall rules;
- verify Docker port mappings;
- expose only the sensor ports you intentionally want public.

## Website sidecar trap paths do not work

Check:

```bash
curl -i https://your-domain.example/.env
curl -i https://your-domain.example/wp-login.php
```

Expected: normal homepage still reaches the real website, while trap-path requests appear in the HoneyPot dashboard.

Fix:

- confirm reverse-proxy location/matcher order;
- confirm the HoneyPot HTTP sensor upstream address;
- pass `X-Real-IP` and `X-Forwarded-For`;
- do not route paths used by real users or real admins to the honeypot.

## Alerts do not fire

Check provider status and send a test from the dashboard or API.

Common fixes:

- `HONEYPOT_ALERTS_ENABLED=true` is not enough by itself;
- Slack/Discord/n8n need webhook URLs;
- Telegram needs both bot token and chat ID;
- SMTP needs host, credentials/from/to, and TLS/SSL settings;
- real delivery is only proven after a successful test message.

## n8n webhook inactive

Check:

```bash
docker compose --profile automation ps
docker compose --profile automation logs -f n8n
```

Fix:

- set `N8N_ENCRYPTION_KEY` in `.env`;
- keep n8n private on `127.0.0.1:5678` unless separately protected;
- import/activate workflows with the setup script;
- restart n8n after activation if webhooks are not registered.

## Database or log permission errors

Docker checks:

```bash
docker volume inspect honeypotv203_honeypot_data || true
docker volume inspect honeypotv203_honeypot_logs || true
docker compose logs honeypot
```

Local checks:

```bash
python - <<'PY'
from pathlib import Path
for p in ['honeypot.db', 'honeypot.log']:
    path = Path(p)
    print(p, 'exists=', path.exists(), 'parent_writable=', path.parent.exists())
PY
```

Fix: use Docker named volumes or local paths writable by the current user.

## Production startup refuses to run

HoneyPot v3 fails closed in production when secrets or bootstrap settings are unsafe.

Fix:

```bash
python setup.py
```

Then verify that `.env` has a strong `HONEYPOT_AUTH_SECRET`, strong admin credentials, and `HONEYPOT_ALLOW_DEFAULT_ADMIN=false`.
