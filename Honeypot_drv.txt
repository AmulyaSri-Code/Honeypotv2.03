Critical/High findings:
1. Production secret enforcement is only a warning, not fail-closed.
   Files: security.py, api.py
   Why it matters: if HONEYPOT_AUTH_SECRET is missing, tokens use change-me-in-production. api.py reports a warning but still runs.
   Suggested fix: in production mode, refuse startup if HONEYPOT_AUTH_SECRET is empty/default.

2. Local default admin behavior can recreate/refresh admin/admin when env credentials are missing.
   File: api.py bootstrap_admin()
   Why it matters: safe for local testing only, dangerous if env loading breaks or the dashboard becomes reachable.
   Suggested fix: allow admin/admin only in explicit local/dev mode; production should fail startup.

3. UI and docs advertise default login admin/admin.
   Files: dashboard/index.html, README.md, tests/test_dashboard_ui.py
   Why it matters: encourages weak operational habits and gives attackers a first credential pair.
   Suggested fix: hide default credentials unless HONEYPOT_ENV=development or replace with setup instructions.

4. Alerting configuration is incomplete.
   Current config check: HONEYPOT_ALERTS_ENABLED=true, TELEGRAM_BOT_TOKEN set, TELEGRAM_CHAT_ID empty, Slack/Discord empty, N8N webhook set.
   Why it matters: operators may think alerts are active while Telegram cannot deliver.
   Suggested fix: dashboard/deployment check should flag provider-level missing fields.

5. Deployment check does not validate secret strength.
   File: api.py _deployment_checks()
   Why it matters: HONEYPOT_ADMIN_PASS of short length can pass if it is not one of known defaults.
   Suggested fix: require minimum length/entropy for admin password and auth secret in production.

Good signs:
- Sensitive telemetry endpoints require authentication in tests and code.
- SQL queries use parameters for attacker-controlled values.
- Captured command text is sanitized/truncated before DB/log storage.
- X-Forwarded-For is ignored unless remote address is configured as trusted proxy.
- Security headers are applied globally.
- Query limits are clamped.

2. Gate admin/admin defaults behind explicit development mode.
3. Improve deployment checks to validate provider-specific alert readiness.
4. Remove default login text from dashboard for production.
5. Add tests for all above.
