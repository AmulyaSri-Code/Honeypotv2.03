Project mind map

HoneyPot v3

A professional map of the honeypot platform: runtime, dashboard, security posture, alerting, optional n8n automation, testing, repository hygiene, and future roadmap.

HoneyPot v3 Capture suspicious traffic → classify events → alert analysts → optionally automate response with n8n

Mission

  • Deception-first defense lab
  • Safe capture of suspicious activity
  • Analyst-ready event intelligence
  • Simple local-first deployment

Core Runtime

  • Flask app and API routes
  • Dashboard serving
  • Honeypot event capture
  • Environment loader with safe defaults
  • Health and metadata endpoints

Dashboard

  • HoneyPot v3 Defense Console
  • Login modal and browser UI
  • Threat and health visibility
  • Alert status: Slack, Telegram, Discord, n8n
  • Zero-console-error smoke verified

Security

  • Loopback-first bindings
  • Token-based API auth
  • No exposed webhook or token values
  • Runtime artifacts excluded from commits
  • n8n local UI bound to 127.0.0.1

Alerting and n8n Automation

  • Provider status exposes booleans only
  • Slack, Telegram, Discord, and n8n provider paths
  • N8N_WEBHOOK_URL sends structured payloads
  • Payload includes source, summary, and original event
  • Optional Docker Compose automation profile
  • Sample workflow: critical alert router

Configuration and Deployment

  • .env.example with safe placeholders
  • setup.py interactive and CLI setup
  • docker-compose.yml for app plus optional automation
  • Local n8n command: docker-compose --profile automation up -d n8n
  • README documents quick start, alerts, Docker, and deployment checklist

Machine Learning

  • Attack classifier module
  • Model and vectorizer artifacts
  • Threat summary support
  • Classifier behavior tests

Testing

  • Pytest suite: API, alerts, dashboard, safety, ML, setup
  • Python compile checks
  • n8n JSON workflow validation
  • Dashboard JS syntax check
  • Docker Compose config validation
  • API and browser smoke checks

Repository Hygiene

  • Source, docs, tests, and workflow committed
  • Generated runtime files stay local
  • Examples: honeypot.db, honeypot.log, reports, caches
  • Main branch pushed to origin

Website Backend Integration

  • Run HoneyPot v3 as a private sidecar beside the real website backend
  • Route only suspicious trap paths such as /.env, /.git, /wp-login.php, and /phpmyadmin
  • Includes Nginx, Caddy, Node.js/Express, Python, and Docker Compose examples
  • Keep the dashboard private behind localhost, VPN, SSO, or IP allowlisting
  • Forward X-Real-IP and X-Forwarded-For for useful telemetry

Future Roadmap

  • IP reputation and ASN enrichment
  • Ticketing and case-management workflows
  • Daily/weekly n8n reports
  • More dashboard analytics
  • Hardened production deployment guide
